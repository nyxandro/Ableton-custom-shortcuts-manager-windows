Список исправлений:

v0.2.1
- Исправлена проблема, когда при раскрытии программы из трея возникала ошибка.

v0.2
- Заменены названия колонок и кнопок:
Ableton default -> Key
Custom -> Custom Key
Add Shortcut -> Add
Delete Shortcut -> Delete
Нет совпадений -> No matches found

- удалены шорткаты, основанные на drag and drop, оставляем только конкретные команды нажатий кнопок;
- удалены дублирующиеся дефолтные и состоящие из одной кнопки (например Up/Down arrow keys) шорткаты в списке всех шорткатов;
- цвет логотипа Ableton в шапке программы сделал серым, чтобы на темных темах был виднее;
- убрал цветовое оформление кнопок add, delete и reset all так как стандартные кнопки во всех темах по-разному раскрашивались, решено оставить базовый цвет системы;
- при нажатии кнопки + add (добавить сопоставление шортката) в списке шорткатов выбирается именно тот, на котором стоит выделение в таблице шорткатов, раньше это не учитывалось и просто выбирался первый шорткат из списка, что вызывало путаницу;
- добавлена кликабельная ссылка снизу программы, ведущая на https://t.me/abletonliveusers
- когда возникает конфликт сопоставления шорткатов (когда кастомный шорткат перекрывает дефолтный, а дефольтный ничем не переназначен) при клике на заначок конфликта можно посмотреть подсказку, с каким именно шорткатом происходит конфликт;
- добавлен автозахват шорткатов как через окно добавление шортката, так и через окно редактирование, оставил кнопку повторного переназначения, если ошиблись во время ввода.
- захват шорткатов специально ограничил только с модификаторами Ctrl, Shift, Alt, на остальные поставил запрет чтобы не было конфликтов.

v0.1

- добавлена настройка запуска в свернутом режиме в трей;
- добавлена настройка предотвращения закрытия окна программы, вместо этого она сворачивается в трей, полностью закрыть можно через ПКМ по иконке программы в трее;
